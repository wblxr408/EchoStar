## 6.4 Code Review 机制

### 代码评审规则

- 所有代码必须经过 Code Review 才能合并到 `main` 分支
- Reviewer: 架构师 + 模块负责人
- 评审重点:
  - 是否符合模块边界
  - 是否有安全漏洞 (SQL 注入 / XSS)
  - 是否有性能问题
  - 代码可读性

### Git 工作流

```
main (受保护,不允许直接 push)
  ↑
  │ PR (需要 Code Review)
  │
feature/story-publish (功能分支)
```

## 6.6 风险与应急预案

| 风险                         | 概率 | 应急方案                            |
| ---------------------------- | ---- | ----------------------------------- |
| **PostGIS 性能不达标** | 中   | 降级为简单距离计算,牺牲精度         |
| **高德地图配额不足**   | 低   | 切换到 OpenStreetMap                |
| **团队成员请假**       | 中   | 架构师顶上,延期 1 周                |
| **第三方服务故障**     | 低   | OSS 切换本地存储,OAuth 切换邮箱登录 |
