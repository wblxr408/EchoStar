## 3.2 表结构设计

### 3.2.1 users 表

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE,
    password_hash VARCHAR(255),           -- 邮箱注册时需要
    oauth_provider VARCHAR(20),           -- 'github' / null
    oauth_id VARCHAR(100),                -- GitHub 用户 ID
    avatar_url VARCHAR(500),
    role VARCHAR(20) DEFAULT 'user',      -- 'user' / 'admin'
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_oauth ON users(oauth_provider, oauth_id);
```

**字段说明**:

| 字段               | 类型    | 说明                             |
| ------------------ | ------- | -------------------------------- |
| `password_hash`  | VARCHAR | 仅邮箱注册需要,使用 bcrypt 加密  |
| `oauth_provider` | VARCHAR | OAuth 登录来源,目前只有 'github' |
| `role`           | VARCHAR | 'user' 普通用户 / 'admin' 管理员 |

### 3.2.2 stories 表 (核心表)

```sql
-- 必须先安装 PostGIS 扩展
CREATE EXTENSION IF NOT EXISTS postgis;

CREATE TABLE stories (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    images JSONB DEFAULT '[]',            -- ["url1", "url2", ...]
    location GEOGRAPHY(Point, 4326) NOT NULL,  -- PostGIS 地理坐标
    emotion_tag VARCHAR(20),              -- '治愈'/'难过'/'开心'/'打卡'
    is_time_capsule BOOLEAN DEFAULT FALSE,
    unlock_at TIMESTAMP,                  -- 时光胶囊解锁时间
    visibility VARCHAR(20) DEFAULT 'public',  -- 'public'/'shadowban'/'deleted'
    view_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 关键索引 (性能核心!)
CREATE INDEX idx_stories_location ON stories USING GIST(location);  -- 空间索引
CREATE INDEX idx_stories_created_at ON stories(created_at DESC);
CREATE INDEX idx_stories_user_id ON stories(user_id);
CREATE INDEX idx_stories_visibility ON stories(visibility);
CREATE INDEX idx_stories_unlock_at ON stories(unlock_at) WHERE is_time_capsule = TRUE;
```

**字段说明**:

| 字段            | 类型      | 说明                       | 示例                                                      |
| --------------- | --------- | -------------------------- | --------------------------------------------------------- |
| `content`     | TEXT      | 故事正文                   | "今天心情很好..."                                         |
| `images`      | JSONB     | 图片 URL 数组              | `["https://oss.../1.jpg"]`                              |
| `location`    | GEOGRAPHY | **PostGIS 地理坐标** | `POINT(116.4074 39.9042)`                               |
| `emotion_tag` | VARCHAR   | 情绪标签                   | '治愈' / '难过' / '开心' / '打卡'                         |
| `visibility`  | VARCHAR   | 可见性                     | 'public' 正常 / 'shadowban' 仅自己可见 / 'deleted' 软删除 |

**PostGIS 字段解释**:

```sql
-- location 字段使用 GEOGRAPHY 类型,坐标系 SRID 4326 (WGS84)
-- 插入数据示例:
INSERT INTO stories (user_id, content, location, ...)
VALUES (
  1,
  '今天去了故宫',
  ST_GeogFromText('POINT(116.4074 39.9042)'),  -- 经度在前,纬度在后!
  ...
);

-- 范围查询示例 (查询 1km 范围内的故事):
SELECT * FROM stories
WHERE ST_DWithin(
  location,
  ST_GeogFromText('POINT(116.4074 39.9042)'),
  1000  -- 单位:米
)
AND visibility = 'public';
```

### 3.2.3 admin_actions 表

```sql
CREATE TABLE admin_actions (
    id SERIAL PRIMARY KEY,
    story_id INTEGER REFERENCES stories(id) ON DELETE CASCADE,
    admin_id INTEGER NOT NULL REFERENCES users(id),
    action_type VARCHAR(20) NOT NULL,     -- 'recommend' / 'shadowban' / 'restore'
    reason TEXT,                          -- 操作理由
    created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_admin_actions_story ON admin_actions(story_id);
CREATE INDEX idx_admin_actions_admin ON admin_actions(admin_id);
```

**action_type 枚举值**:

| 值            | 含义                  |
| ------------- | --------------------- |
| `recommend` | 设为推荐 (首页展示)   |
| `shadowban` | 仅自己可见 (隐形封禁) |
| `restore`   | 恢复正常              |

## 3.3 Sequelize 模型定义

### User 模型

```javascript
// src/modules/auth/auth.model.js
import { DataTypes } from 'sequelize';
import sequelize from '../../config/database.js';

const User = sequelize.define('User', {
  id: {
    type: DataTypes.INTEGER,
    primaryKey: true,
    autoIncrement: true
  },
  username: {
    type: DataTypes.STRING(50),
    allowNull: false,
    unique: true
  },
  email: {
    type: DataTypes.STRING(100),
    unique: true
  },
  passwordHash: {
    type: DataTypes.STRING(255),
    field: 'password_hash'
  },
  oauthProvider: {
    type: DataTypes.STRING(20),
    field: 'oauth_provider'
  },
  oauthId: {
    type: DataTypes.STRING(100),
    field: 'oauth_id'
  },
  avatarUrl: {
    type: DataTypes.STRING(500),
    field: 'avatar_url'
  },
  role: {
    type: DataTypes.STRING(20),
    defaultValue: 'user'
  }
}, {
  tableName: 'users',
  underscored: true,
  timestamps: true
});

export default User;
```

### Story 模型

```javascript
// src/modules/story/story.model.js
import { DataTypes } from 'sequelize';
import sequelize from '../../config/database.js';

const Story = sequelize.define('Story', {
  id: {
    type: DataTypes.INTEGER,
    primaryKey: true,
    autoIncrement: true
  },
  userId: {
    type: DataTypes.INTEGER,
    allowNull: false,
    field: 'user_id'
  },
  content: {
    type: DataTypes.TEXT,
    allowNull: false
  },
  images: {
    type: DataTypes.JSONB,
    defaultValue: []
  },
  location: {
    type: DataTypes.GEOGRAPHY('POINT', 4326),  // PostGIS 类型
    allowNull: false
  },
  emotionTag: {
    type: DataTypes.STRING(20),
    field: 'emotion_tag'
  },
  isTimeCapsule: {
    type: DataTypes.BOOLEAN,
    defaultValue: false,
    field: 'is_time_capsule'
  },
  unlockAt: {
    type: DataTypes.DATE,
    field: 'unlock_at'
  },
  visibility: {
    type: DataTypes.STRING(20),
    defaultValue: 'public'
  },
  viewCount: {
    type: DataTypes.INTEGER,
    defaultValue: 0,
    field: 'view_count'
  }
}, {
  tableName: 'stories',
  underscored: true,
  timestamps: true
});

// 关联关系
Story.belongsTo(User, { foreignKey: 'userId' });
User.hasMany(Story, { foreignKey: 'userId' });

export default Story;
```

## 3.4 常用查询示例

### 范围查询 (1km 内的故事)

```javascript
import { sequelize } from '../../config/database.js';
import Story from './story.model.js';

const nearbyStories = await Story.findAll({
  where: sequelize.literal(`
    ST_DWithin(
      location,
      ST_GeogFromText('POINT(${lng} ${lat})'),
      ${radius}
    )
    AND visibility = 'public'
  `),
  limit: 100
});
```

### 同地点故事墙 (100米内)

```javascript
const locationWall = await Story.findAll({
  where: sequelize.literal(`
    ST_DWithin(
      location,
      ST_GeogFromText('POINT(${lng} ${lat})'),
      100
    )
    AND visibility = 'public'
  `),
  order: [['created_at', 'DESC']],
  limit: 50
});
```

### 计算距离

```javascript
const stories = await sequelize.query(`
  SELECT
    *,
    ST_Distance(
      location,
      ST_GeogFromText('POINT(${lng} ${lat})')
    ) AS distance
  FROM stories
  WHERE visibility = 'public'
  ORDER BY distance
  LIMIT 20
`, { type: QueryTypes.SELECT });
```

## 3.6 性能优化建议

### 索引策略

| 查询场景         | 索引类型      | 索引定义                                                       |
| ---------------- | ------------- | -------------------------------------------------------------- |
| 空间范围查询     | GIST 空间索引 | `CREATE INDEX ... USING GIST(location)`                      |
| 按时间排序       | B-Tree 索引   | `CREATE INDEX ... ON created_at DESC`                        |
| 用户故事列表     | B-Tree 索引   | `CREATE INDEX ... ON user_id`                                |
| 时光胶囊定时任务 | 部分索引      | `CREATE INDEX ... ON unlock_at WHERE is_time_capsule = TRUE` |

### 查询优化

```sql
-- 好的查询 (使用空间索引)
EXPLAIN ANALYZE
SELECT * FROM stories
WHERE ST_DWithin(location, ST_GeogFromText('POINT(116 39)'), 1000)
AND visibility = 'public';

-- 不好的查询 (无法使用索引)
SELECT * FROM stories
WHERE ST_Distance(location, ST_GeogFromText('POINT(116 39)')) < 1000;
```
