## 5.2 性能目标与测试方法

### 5.2.1 性能指标

| 指标         | 目标值      | 测试方法                       | 验收标准              |
| ------------ | ----------- | ------------------------------ | --------------------- |
| API 响应时间 | P95 < 500ms | Postman / Locust 压测          | 95% 请求 < 500ms      |
| LBS 空间查询 | < 200ms     | PostgreSQL `EXPLAIN ANALYZE` | 1km 范围查询 < 200ms  |
| 地图首屏加载 | < 3s        | Chrome Lighthouse              | Performance 分数 > 70 |
| 并发支持     | 1000 QPS    | 阿里云 PTS 压测                | 无 5xx 错误           |
| 图片上传     | < 5s (1MB)  | 实际测试                       | 成功率 > 99%          |

### 5.2.2 压测场景

**场景 1: 地图范围查询压测**

```bash
# 使用 Locust 压测工具
# locustfile.py
from locust import HttpUser, task

class MapUser(HttpUser):
    @task
    def explore(self):
        self.client.get("/map/explore?lat=39.9&lng=116.4&radius=1000")

# 执行: locust -f locustfile.py --host=http://localhost:3000
# 模拟 1000 并发用户
```

**预期结果**:

- P95 响应时间 < 500ms
- CPU 使用率 < 80%
- 数据库连接数 < 100

## 5.3 技术债务管理

### 5.3.1 MVP 阶段可接受的简化

**现在可以做的简化** (不影响核心功能):

- 不做分库分表 (10 万数据内单表足够)
- 不做消息队列 (定时任务用 node-cron)
- 不做服务拆分 (单体更快)
- 不做 CDN (用户少)
- 不做全文搜索 (用 PostgreSQL `LIKE`)
- 不做 Redis 集群 (单机够用)

### 5.3.2 必须在 V2.0 前解决

**技术债务清单**:

1. **图片清理机制**: 删除的故事对应的图片未清理,会占用 OSS 空间

- **解决方案**: 定时任务扫描 7 天前删除的故事,清理 OSS 图片

2. **数据库备份**: 目前无自动备份

- **解决方案**: 配置 PostgreSQL `pg_dump` 定时备份到 OSS

3. **监控告警**: 无法及时发现故障

   - **解决方案**: 接入 Sentry (错误监控) + Prometheus (性能监控)
4. **日志收集**: 日志分散在多台服务器

   - **解决方案**: 使用 Winston + Loki 集中收集

## 5.5 安全风险

| 风险                | 攻击方式             | 应对方案                         |
| ------------------- | -------------------- | -------------------------------- |
| **SQL 注入**  | 恶意构造查询参数     | 使用 Sequelize ORM 参数化查询    |
| **XSS 攻击**  | 故事内容插入恶意脚本 | 前端使用 `DOMPurify` 过滤 HTML |
| **CSRF 攻击** | 伪造请求             | JWT Token 验证                   |
| **暴力破解**  | 疯狂尝试登录         | 限流 10 次/分钟                  |
| **图片炸弹**  | 上传超大图片         | 前端限制 1MB,OSS 限制 5MB        |
| **坐标伪造**  | 伪造 GPS 坐标        | 暂不验证 (MVP 不做)              |

## 5.6 故障应急预案

### 5.6.1 数据库故障

**症状**: API 返回 5001 数据库错误

**应急措施**:

1. 检查 PostgreSQL 服务状态: `systemctl status postgresql`
2. 查看慢查询日志: `tail -f /var/log/postgresql/slow.log`
3. 重启服务: `systemctl restart postgresql`
4. 从备份恢复 (如果数据损坏)

---

### 5.6.2 OSS 上传失败

**症状**: 用户反馈图片上传失败

**应急措施**:

1. 检查 OSS 控制台是否欠费
2. 检查 STS Token 是否过期
3. 临时切换到服务器本地存储 (降级方案)

---

### 5.6.3 地图无法加载

**症状**: 地图白屏或报错

**应急措施**:

1. 检查高德 API Key 是否过期
2. 检查是否超出配额
3. 临时切换到 OpenStreetMap (备选方案)
