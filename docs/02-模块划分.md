## 2.3 模块职责定义

### Auth 模块

**职责**:

- 邮箱注册/登录
- GitHub OAuth 登录
- JWT Token 签发与验证
- 用户信息管理

**对外接口**:

```javascript
// auth.service.js
export const AuthService = {
  register(email, password)      // 注册
  login(email, password)         // 登录
  loginWithGitHub(code)          // GitHub 登录
  verifyToken(token)             // 验证 Token
  getCurrentUser(userId)         // 获取用户信息
}
```

**依赖**:

- 不依赖其他模块
- 被 Story/Admin 模块依赖

---

### Story 模块

**职责**:

- 故事发布 (图文 + 位置)
- 故事查看/删除
- 时光胶囊定时解锁逻辑
- OSS 上传凭证生成

**对外接口**:

```javascript
// story.service.js
export const StoryService = {
  createStory(userId, data)      // 发布故事
  getStoryById(storyId, userId)  // 查看故事 (需判断可见性)
  deleteStory(storyId, userId)   // 删除故事
  getMyStories(userId)           // 我的故事列表
  getUploadToken()               // 获取 OSS 上传凭证
  unlockTimeCapsule(storyId)     // 解锁时光胶囊 (定时任务调用)
}
```

**依赖**:

- 依赖 Auth 模块获取 `userId`
- 不依赖 Map 模块

---

### Map 模块

**职责**:

- LBS 范围查询 (PostGIS `ST_DWithin`)
- 随机漫步算法
- 同地点故事墙
- 点聚合数据返回

**对外接口**:

```javascript
// map.service.js
export const MapService = {
  exploreStories(lat, lng, radius)    // 范围查询
  randomWalk()                        // 随机漫步
  getLocationWall(lat, lng, radius)   // 同地点故事墙
  getClusterData(bounds)              // 获取聚合数据
}
```

**依赖**:

- 只读取 Story 数据
- 不修改 Story 数据
- 不依赖 Auth 模块 (公开查询)

---

### Admin 模块

**职责**:

- 内容策展 (推荐/置顶)
- Shadowban 违规内容
- 举报处理
- 数据统计

**对外接口**:

```javascript
// admin.service.js
export const AdminService = {
  recommendStory(storyId, adminId)    // 设为推荐
  shadowbanStory(storyId, reason)     // Shadowban
  restoreStory(storyId)               // 恢复正常
  getReports()                        // 举报列表
  getStatistics()                     // 数据统计
}
```

**依赖**:

- 依赖 Auth 模块验证管理员身份
- 操作 Story 数据

```

          ▼

     │  Admin  │◄─── 管理模块,操作 Story
     └─────────┘
```

## 2.5 模块通信规则

### 允许的调用

```javascript
// story.service.js 可以调用 auth
import { AuthService } from '../auth/auth.service.js';
const user = await AuthService.getCurrentUser(userId);

// admin.service.js 可以调用 story
import { StoryService } from '../story/story.service.js';
await StoryService.getStoryById(storyId);
```

### 禁止的调用

```javascript
// ❌ story 不能调用 map
import { MapService } from '../map/map.service.js'; // 错误!

// ❌ map 不能调用 admin
import { AdminService } from '../admin/admin.service.js'; // 错误!

// ❌ map 不能直接修改 Story 数据
await Story.update({ ... }); // 错误!应该通过 StoryService
```

## 2.6 代码规范

### 代码层级

```
Controller 层: 处理 HTTP 请求,参数验证,调用 Service
    ↓
Service 层: 业务逻辑,事务处理,调用 Model
    ↓
Model 层: 数据库操作,ORM 定义
```

**示例**:

```javascript
// story.controller.js
export const createStory = async (req, res) => {
  const { content, images, location } = req.body;
  const userId = req.user.id; // 来自 JWT 中间件

  const story = await StoryService.createStory(userId, {
    content,
    images,
    location
  });

  res.json({ code: 0, data: story });
};

// story.service.js
export const createStory = async (userId, data) => {
  // 业务逻辑
  const story = await Story.create({
    userId,
    ...data,
    visibility: 'public'
  });
  return story;
};
```
