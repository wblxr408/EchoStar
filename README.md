# 🚀[PRD & UI] 治愈系地图社区 V1.0 产品需求与视觉交互规范

## 📝 1. 背景与产品愿景 (Product Vision)
本次 PR 提交了 V1.0 MVP 版本的产品需求文档（PRD）与核心 UI 设计规范。

**产品定位**：一个基于 LBS（地理位置服务）的轻量级、治愈系匿名分享平台。
**核心价值**：为当代年轻人提供一个无社交压力的“时空树洞”。在这里，地理坐标是锚点，情感是连接。我们希望通过这款产品，让用户感受到“原来在这个坐标，也有人跟我有一段相似的心事”。

## 🎨 2. UI/UX 设计规范与交互逻辑 (Design System)

> **TO 前端团队**：本项目的视觉核心在于“治愈感”与“轻盈感”，请严格遵循以下 UI 调性进行开发。详细原型图见底部链接。
### 2.0 网站首页可以选择登录界面 加上GitHub项目界面 以及logo ”关于我们“界面 也可以选择地图界面 留言功能登录后解锁

### 2.1 整体视觉基调 (Visual Tone)
*   **主题风格**：治愈、静谧、低压力、微毛玻璃（Glassmorphism）质感。
*   **色彩规范**：
    *   **主色调**：宁静蓝（#8ECAE6） / 治愈青（#219EBC）
    *   **强调色**：落日橘（#FB8500）- 用于突出核心按钮，如“随机漫步”、“发布”
    *   **背景色**：大面积留白或极浅的暖灰色（#F8F9FA）
*   **地图皮肤定制 (⚠️ 重点)**：
    *   **绝对禁止**使用默认的高德/百度导航地图样式（过于生硬）。
    *   前端需调用地图 API 配置**自定义底图**，去掉繁杂的商铺 POI（兴趣点），只保留基础路网和自然景观，颜色调整为“午夜蓝”或“马卡龙灰”等冷暖色调（实现可能比较难，前端界面的调整具体实现再说），突出用户的“故事标点”。

### 2.2 核心页面 UI 布局要求
1.  **全屏地图主页 (Map Home)**
    *   **UI结构**：地图全屏铺满无边框。顶部悬浮半透明搜索/筛选栏；右下角悬浮“当前定位”和“随机漫步”悬浮球（FAB）；正下方为醒目的“+ 发布故事”按钮。
    *   **标点样式 (Marker)**：不同情绪对应不同颜色的圆点呼吸灯；遇到密集坐标时，必须展示为聚合气泡（显示数字）。
2.  **故事预览卡片 (Bottom Sheet Card)**
    *   **交互**：点击地图上的标点后，从屏幕底部**平滑向上弹起（Slide-up）**一张圆角卡片 地图皮肤定制的内容难以实现可以在交互界面 比如查看他人留言时 检索时呈现不同情绪底色。
    *   **UI元素**：左侧/顶部展示封面缩略图，右侧/下方展示摘要（最多3行）、发布时间（如“2小时前”）、距离（如“相距 1.2km”）。
3.  **时光胶囊状态 (Time Capsule)**
    *   **视觉**：未解锁的胶囊，地图上的标点显示为**圆圈进度条**。点击后卡片不可见正文，仅展示一个沙漏动效和倒计时文案（如：“距离解封还有 3天”）。同时和“定期销毁留言”功能一起，可以分别呈现为正向和反向的圆圈进度条。

### 2.3 关键交互动效 (Animations)
*   **飞行漫步 (FlyTo)**：点击“随机漫步”时，地图视角必须有“拉高缩放 -> 跨城市平移 -> 俯冲降落”的三段式平滑过渡动画，营造极强的浪漫感。
*   **弹窗出现**：所有弹窗（发布、详情页）建议带有底层毛玻璃遮罩（`backdrop-filter: blur(10px)`），体现轻量感。

## 🗺️ 3. 核心用户故事 (User Stories)

### 👤 用户端视角 (C-End)
*   **低压倾诉**：作为用户，我可以在当前坐标发布匿名图文，它不仅是记录，更是一种情绪的安全宣泄。
*   **时空共鸣**：作为读者，我可以查看“同地点故事墙”，看到过去一年里这个坐标发生过的其他故事。
*   **制造惊喜**：作为探索者，我可以通过“随机漫步”飞到未知的远方，或埋下一个“时光胶囊”给未来的自己。

### 🛡️ 管理员视角 (B-End - 守夜人)
*   **内容策展**：作为管理员，我能在后台将优质的高清图文设置为“首页推荐”，保证新用户一打开地图就能看到美好的内容，确立社区调性。
*   **隐形守护**：对于发广告/违规的用户，管理员执行“仅自己可见（Shadowban）”操作，优雅地隔离戾气，保护社区纯净度。
