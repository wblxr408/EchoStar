## 问题

### 1. 修改个人信息出现错误 | @后端 ：@解决
- 现象：
- backend\tests\test-results\request-records下的auth模块的测试记录， 序号9。
- 原因：语法错误，auth.service.js:395 使用了 require：
id: { [require('sequelize').Op.ne]: userId }  // ES Module 中不能用 require
原因：项目使用 ES Module（import/export），但在代码中错误地使用了 CommonJS 的 require。
- 解决方案：
在文件顶部检查是否已导入 Op
将 [require('sequelize').Op.ne] 改为 [Op.ne]
