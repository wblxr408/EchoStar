## 问题

### 1. 变量命名问题 | @后端 @解决
- **现象**：接口返回值各字段键名不明确，难以解读，且存在嵌套结构内外层有相同变量名。形如：
```json
{
  "code": 0,
  "data": {
    "favorites": [
      {
        "id": 2,    // 这个是故事独立的收藏 id 还是用户自己收藏夹的 id？
        "createdAt": "2026-03-19T09:16:50.181Z",
        "user": {
          "id": 1,  // 与外层 id 重复，可能给前端带来解读困难
          "username": "favorite_test_user_1773911809960",
          "avatar": null
        }
      }
    ],
    "pagination": {
      "total": 1,
      "page": 1,        // 不明含义
      "limit": 10,      // 不明含义
      "totalPages": 1
    }
  }
}
```
- **解决方案**：如果前端能接受，可以不管。建议明确变量含义，方便所有成员阅读。

### 2. 不存在的故事收藏列表返回值问题 | @后端 @解决
- **现象**：
  - `http://localhost:3000/api/favorites/story/999999`
  - `http://localhost:3000/api/favorites/999999/check`
  - `http://localhost:3000/api/favorites/999999/count`
  - `http://localhost:3000/api/favorites/check-multiple`（请求体包含不存在故事时可能需要更多逻辑以使得存在故事正常返回，不存在故事返回错误信息。而返回字段的形式，具体可能需要与前端商讨）
- 同于评论问题，不存在的故事访问各种数据均返回空列表而非返回错误（故事不存在）信息：
```json
{
  "code": 0,
  "data": {
    "favorites": [],
    "pagination": {
      "total": 0,
      "page": 1,
      "limit": 10,
      "totalPages": 0
    }
  }
}
```
