## 问题

### 1. 不存在故事点赞列表返回值问题 | @后端 @解决
- **现象**：
  - `http://localhost:3000/api/likes/story/999999`
  - `http://localhost:3000/api/likes/999999/check`（应该返回"故事不存在"而不是"未点赞"）
  - `http://localhost:3000/api/likes/999999/count`（应该返回"故事不存在"而不是点赞数）
  - `http://localhost:3000/api/likes/check-multiple`（对于返回列表中不存在的故事，应该返回不存在故事的错误信息而不是点赞数为 0）
- 返回空列表而非错误信息。Like 模块的其他接口已经规避这个问题，会返回错误信息，因此这些接口以及其他模块的接口应该统一：在不存在故事的数据被请求访问时返回错误信息，而非空列表。
```json
{
  "code": 0,
  "data": {
    "likes": [],
    "pagination": {
      "total": 0,
      "page": 1,
      "limit": 10,
      "totalPages": 0
    }
  }
}
```
- **解决方案**：统一接口返回逻辑，当请求的故事不存在时返回错误信息。

### 2. 返回值冗余 | @后端 @解决
- **现象**：
  - `http://localhost:3000/api/likes/1/count`
- 返回值中包含 storyId 字段，而该字段是请求参数，在请求结果中并不需要，应该删除。（如果删除代价大，可以不管，不影响功能）
```json
{
  "code": 0,
  "data": {
    "storyId": 1,
    "likeCount": 1
  }
}
```
- **解决方案**：移除返回值中冗余的 storyId 字段。
