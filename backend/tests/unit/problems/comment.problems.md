## 问题

### 1. 不存在故事请求评论列表返回值问题 | @后端 @产品经理 @商讨 @解决
- **现象**：`http://localhost:3000/api/comments/story/999999` 访问不存在的故事返回一个空评论列表以及统计信息，这与访问存在但没有评论的故事的评论列表有一致的返回值，前端难以判断是故事不存在还是故事存在但没有评论：
```json
{
  "code": 0,
  "data": {
    "comments": [],
    "pagination": {
      "total": 0,
      "page": 1,
      "limit": 10,
      "totalPages": 0
    }
  }
}
```
- **解决方案**：待后端与产品经理商讨是否更改，因为并不太影响功能。若要修改，建议请求评论列表时先判断是否存在 id 为传入参数的故事，不存在时返回错误信息，存在时才返回评论列表。

### 2. 不存在故事请求评论数量统计返回值问题 | @后端 @产品经理 @前端 @商讨 @解决
- **现象**：`http://localhost:3000/api/comments/999999/count` 同于第一个问题，请求不存在故事时返回数量为 0 的统计，这与访问存在但没有评论的故事的评论数量统计是一致的，前端难以判断是故事不存在还是故事存在但没有评论：
```json
{
  "code": 0,
  "data": {
    "storyId": "999999",
    "commentCount": 0
  }
}
```
- **解决方案**：待后端与产品经理商讨是否更改，因为并不太影响功能。若要修改，建议请求评论数量统计时先判断是否存在 id 为传入参数的故事，不存在时返回错误信息，存在时才返回评论数量统计。
