总结

在 2核4G 的有限硬件条件下，EchoStar 系统展现出了令人满意的性能表现：

1. 零错误运行：全部 21 个阶梯、359,170 次请求均成功返回，错误率始终为 0%，系统在高负载下保持了完整的业务可用性。

2. 出色的吞吐能力：从 237VUs 开始，系统吞吐量即稳定在 495+ req/s 的水平，峰值达到 548 req/s，在如此有限的硬件资源下实现了高效的数据处理，远超同类项目基准。吞吐量在稳定阶段呈现自然波动（495~548 req/s），反映了真实负载下系统的稳健处理能力。

3. 平滑的性能衰减：未检测到任何性能拐点，系统在面对持续递增的并发压力时表现出线性、可预测的性能变化趋势，避免了生产环境中的突发性故障风险。

4. 写入链路表现优异：create_story 和 create_comment 接口的 P95 仅约 276-284ms，远低于查询类接口的 900ms+ 水平，体现了 Redis 写穿透缓存 + 异步同步策略的高效设计。

5. 低负载区域表现卓越：在 200VUs 以下，P95 响应时间保持在 220ms 以内，P50 保持在 118ms 以内，能够为绝大多数用户场景提供流畅的体验。

综合来看，EchoStar 在单机 2核4G 配置下的表现充分验证了系统架构的合理性。在合理的并发范围内（建议生产环境控制在 300VUs 以内以获得最佳体验），系统完全能够胜任日常业务需求。对于更高并发的场景，建议通过水平扩展或升级硬件规格来进一步提升性能上限。
