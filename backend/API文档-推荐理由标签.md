# 推荐理由标签 - 前端对接文档

## 概述

故事详情页新增 `recommendationTags` 字段，以彩色标签（Tag）形式展示"为什么推荐这个故事"。最多展示 **3 个**标签，标签数量为 0 时不展示。

---

## 接口变更

### GET /api/stories/:id

**新增请求参数（query）：**

| 参数   | 类型     | 必填 | 说明                                                    |
| ------ | -------- | ---- | ------------------------------------------------------- |
| `lng`  | `number` | 否   | 用户当前经度，传入后可触发"附近热议"标签                 |
| `lat`  | `number` | 否   | 用户当前纬度，传入后可触发"附近热议"标签                 |

**调用示例：**

```text
# 有用户坐标（推荐，能触发全部标签）
GET /api/stories/1893456789012345678?lng=116.4034&lat=39.9150

# 无用户坐标（"附近热议"标签不会出现）
GET /api/stories/1893456789012345678
```

> 建议前端在已登录且能获取到用户位置时才传 `lng`/`lat`。未登录时"此刻共鸣"和"附近热议"不会出现。

---

## 响应数据变更

在原有 `data` 对象中新增一个字段：

| 字段               | 类型                        | 说明                                       |
| ------------------ | --------------------------- | ------------------------------------------ |
| `recommendationTags` | `Array<RecommendationTag>` | 推荐理由标签数组，最多 3 个。空数组表示无标签 |

### RecommendationTag 对象

| 字段  | 类型     | 说明                     |
| ----- | -------- | ------------------------ |
| `tag` | `string` | 标签中文显示文本         |
| `type` | `string` | 标签英文标识，用于样式区分 |

---

## 响应示例

```json
{
  "code": 0,
  "data": {
    "id": "1893456789012345678",
    "content": "今天在公园遇到了一只很可爱的猫...",
    "emotionTag": "开心",
    "isRecommended": true,
    "viewCount": 1234,
    "likeCount": 56,
    "commentCount": 12,
    "favoriteCount": 8,
    "recommendationTags": [
      { "tag": "编辑精选", "type": "editor_pick" },
      { "tag": "此刻共鸣", "type": "mood_resonance" },
      { "tag": "附近热议", "type": "trending_nearby" }
    ],
    "author": { "id": 1, "username": "示例用户", "avatar": "...", "vip": 0 }
  }
}
```

---

## 各标签说明

| `type` 值         | `tag` 文本 | 后端判定逻辑                                           | 建议配色       |
| ----------------- | ---------- | ------------------------------------------------------ | -------------- |
| `editor_pick`     | 编辑精选   | 故事被管理员标记为精选（`isRecommended = true`）       | 金色 / 琥珀色  |
| `mood_resonance`  | 此刻共鸣   | 用户最近发布故事中最常见的情绪标签与当前故事一致        | 蓝紫色 / 渐变  |
| `trending_nearby` | 附近热议   | 用户在故事 150km 内 + 故事发布于 7 天内 + 互动数 ≥ 5   | 橙红色 / 暖色  |

---

## 各种场景下的返回值

| 场景                                  | `recommendationTags` 值                                                         |
| ------------------------------------- | ------------------------------------------------------------------------------- |
| 查看自己的故事                        | `[]`（空数组，永远不展示推荐标签）                                              |
| 游客访问 + 故事非精选                 | `[]`                                                                           |
| 游客访问 + 故事是精选                 | `[{ "tag": "编辑精选", "type": "editor_pick" }]`                              |
| 已登录 + 未传坐标 + 情绪匹配+精选     | `[{ "tag": "编辑精选", "type": "editor_pick" }, { "tag": "此刻共鸣", "type": "mood_resonance" }]` |
| 已登录 + 传了坐标 + 附近热议+精选     | `[{ "tag": "编辑精选", "type": "editor_pick" }, { "tag": "附近热议", "type": "trending_nearby" }]`  |

---

## 前端展示规则

1. `recommendationTags` 为空数组时，**不展示任何标签**
2. 最多同时展示 3 个标签
3. 标签位置建议放在故事内容上方或作者信息下方
4. 不同 `type` 使用不同配色，以视觉区分推荐理由
